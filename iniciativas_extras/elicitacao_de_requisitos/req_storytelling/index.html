    Requisitos elicitados referentes a storytelling [1]

    Introdução

    A partir do storytelling realizado pela equipe para validar e aprimorar ideias sobre o produto proposto, foi possível elicitar os requisitos documentados na tabela a seguir.

    Em seguida, foi realizada a priorização no método MoSCOW, que busca auxiliar no processo de síntese dos requisitos elicitados [2].

    Requisitos elicitados

    Código Descrição Prioridade Rastro
    ST01 O sistema deve ser inovador Must have Storytelling 1 - Storytelling 2
    ST02 As mesas dos shoppings devem conter QRCodes Must have Storytelling 1 - Storytelling 2
    ST03 O sistema deve escanear o QRCode Must have Storytelling 1 - Storytelling 2
    ST04 O sistema deve ser Web Must have Storytelling 1 - Storytelling 2
    ST05 O QRCode lideo deve direcionar o usuário para o QRComer Must have Storytelling 1 - Storytelling 2
    ST06 O sistema deve possuir uma área de cadastro do usuário Must have Storytelling 1
    ST07 O sistema deve possuir integração com outras API's de cadastro Should have Storytelling 1
    ST08 O sistema deve obrigar o usuário a cadastrar o cartão de crédito Should have Storytelling 1
    ST09 O sistema deve possuir uma tela de carregamento como feedback para o usuário Could have Storytelling 1
    ST10 O sistema deve informar o shopping onde o usuário está localizado Must have Storytelling 1 - Storytelling 2
    ST11 O sistema deve demonstrar a lista de restaurantes do shopping Must have Storytelling 1
    ST12 O sistema deve permitir a escolha de um restaurante pelo usuário Must have Storytelling 1
    ST13 O sistema deve informar os itens do cardápio do restaurante Must have Storytelling 1 - Storytelling 2
    ST14 O sistema deve permitir que o usuário escolha itens do cardápio do restaurante Must have Storytelling 1 - Storytelling 2
    ST15 O sistema deve conter um campo de observações para o usuário detalhar o pedido Could have Storytelling 1
    ST16 O sistema deve permitir a finalização do pedido Must have Storytelling 1 - Storytelling 2
    ST17 O sistema deve permitir a escolha do sistema de pagamento, seja no cartão ou no dinheiro Could have Storytelling 1 - Storytelling 2
    ST18 O sistema deve conter um elemento de segurança para permitir a transação por cartão de crédito Must have Storytelling 1
    ST19 O sistema deve gerar uma senha do pedido do usuário Should have Storytelling 1
    ST20 O sistema deve notificar o usuário de que seu pedido está pronto Should have Storytelling 1
    ST21 O sistema deve evitar extinguir filas Must have Storytelling 1 - Storytelling 2
    ST22 O sistema deve possuir uma área de cadastro do restaurante Must Have Storytelling 2
    ST23 O sistema deve possuir uma tela de cadastro dos produtos do cardápio do restaurante Must Have Storytelling 2
